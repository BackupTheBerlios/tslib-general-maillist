I am certainly not working on the same platform that you are but I have had to deal with these issues using the Hampshire Tsharc Devilray12 resistive touchscreen in my platform.  It also has no pressure sensors.  Hampshire provided me a patch for the file input_raw.c in the plugins directory within tslib1.0.

I also saw this error when running ts_calibrate because TSLIB_TSDEVICE was not set correctly.  It must be set to the actual device that is setup and being used for your touchscreen.  If not set correctly I saw that error.
