Senthilv.Kumar at lntinfotech.com wrote:
> Hi,
> Am working on touch screen issue for Linux 2.6.27 kernel
> 
> i have calibrated touch screen using ts_calibrate utility and i have got 
> updated pointercal file values as follows 
> -4900 60 16865308 -173 6490 -2343568 65536 65536
> 
> Do i need to hard code these values in any touch related file so that 
> touch driver can calibrate the RAW co-ordinates with these calibrated data 
> before sending to application ?
> 

The way we do it is: link our application against tslib and then calibrate each
embedded device at factory init stage and place the calibration result in /etc/pointercal.

This is almost necessary because touchscreen
panels may deviate one from another and hardcoding one calibration value for all
touchscreen panels (even the same mnfr, product, type) is little too
restrictive, IMHO.

> can you please tell me how should i go ahead after touch screen 
> calibration ?

test it. use ts_print or ts_draw to see if you calibrated the screen correctly.
Use the tslib library in your application and it will automagicaly load the calibration
parameters. Reading the fd returned by the ts_fd() should then spew out valid, calibrated
events from your touchscreen device.
