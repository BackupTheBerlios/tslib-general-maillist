I've just discovered this newly maintained tslib library (been using tslib 0.0.2 
from RMK). I see a lot of work was done - congrats.
What I'm interested in is complete static build of tslib library - eg. without 
dynamic loader for plugins. Our embedded platform uses uClibc and only one 
application is running in production environment - statically linked. Since we 
need tslib for our LCD + touchscreen the need for ld and other shared libraries 
on the target rootfs arises.

Has anyone tried to patch/use tslib without the need ld loader? My idea would 
be, that needed plugins would be compiled in libts.a and no plugins are later 
needed on target rootfs.
