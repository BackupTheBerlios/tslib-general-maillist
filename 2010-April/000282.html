Hi,

We are using TSLIB on a TS-TPC-7390 device with a 
SDL/framebuffer application. We had to modify the source to deactivate 
the pressure input (always send 255) cause the hardware don't send it. 
(See http://tech.groups.yahoo.com/group/ts-7390/files/).

We use what seems to be the default settings:

export 
TSLIB_TSEVENTTYPE=H3600

ts.conf:

module_raw inputexport
module pthres pmin=1
module variance delta=30
module dejitter delta=100
module linear

Calibration works fine, cursor don't jitter, everything 
look nice. Our problem is, that sometime when we click too fast, it send us input of the last position clicked with a 2-3pixels difference, 
instead of the real new position which can be completely at the other side of the screen. This 
is awful for a virtual keyboard since some keys duplicate and miss the 
new one.

My guess is one module causing problem, but I have no 
idea which one it could be.
