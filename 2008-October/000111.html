Peter Geoghegan wrote:
> Hinko,
> 
> is --enable-debug=yes just a generic autotools flag that isn't used by
> libts? I ran configure with it as an argument, and the configure
> script reported that config.h was unchanged, and  when I subsequently
> ran make, make reported "nothing to be done for xxx" in all cases.
> 

$ ./configure --enable-debug=yes
Look for:
...
checking whether to enable debugging... yes
...
in configure output (near the end..) to see if it is enabled.

Later on do a :
$ make clean; make; make install

Which will hopefully remove old object files and build new ones 
with debugging enabled.
