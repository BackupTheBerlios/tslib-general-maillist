Peter Geoghegan wrote:
> Hello,
> 
> I'm writing a Linux application that will rely on an Elo touch screen. I
> have it working with X.org on a 2.6 kernel, using proprietary drivers (Elo's
> own 3.2 unified serial driver for Linux).
> 
> I can't seem to find any specific information about getting this device,
> which uses a simple proprietary protocol, to work with Tslib - Google hasn't
> been helpful, and there is no reference to Elo products in ts.conf . This is
> surprising, as there is direct support for it in the Linux kernel (although,
> as I've said, I'm not currently availing of that support).
> 
> If someone could at least point me in the right direction, that would be
> greatly appreciated.

Looking at the elo kernel driver code it seems that the driver registers as 
input device with ABS_X, ABS_Y, ABS_PRESSURE parameters. Which looks like a
valid touchscreen device to me. Try using 'module_raw input' in ts.conf and
look at /proc/bus/input/devices where the device soulhd be registered as a 
eventx. Then run evtest and/or ts_test on that /dev/input/eventx.
