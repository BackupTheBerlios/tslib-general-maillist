When you start the program (eg. ts_calibrate) you should see debugging output on the stderr.

OTOH, using gdb, make sure library and the utility are
compiled with debugging info (eg. compiler flasgs should
look something like .O3 -ggbd3, but certainly not -O0 or -O2;
that would make debugging pretty hard, since compiler
would rearrange the code (optimization)...):

I would recommend:
$ ./configure --enable-debug=yes --enable-static --disable-shared
$ make clean
$ make
$ gdb ./tests/ts_calibrate
