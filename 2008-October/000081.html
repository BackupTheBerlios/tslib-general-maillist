Hi,
I'm having a bit of a problem with getting my gunze touch screen (and
gunze controller) working with tslib. When I try to use the normal
input module I get a "selected device is not a touchscreen I
understand" message, if I instead use module h3600 it seems to work
fine, I can calibrate ok and ts_test works excellent (I've had a few
minutes of fun just drawing on the screen in the test) but when I try
to start it with Xfbdev things get worse. It still works somewhat, it
registers touches at the correct location and I can navigate menus and
such with it, but as soon as I release the screen the cursor jumps
down a bit and to the right (about one third of the screen), I don't
get this behaviour in the test app (then the cursor will stay wherever
it was when I released the screen).
Anyone has any ideas on what  could be wrong? I'm guessing that
perhaps h3600 isn't a perfect match for my screen?

ts.conf:
module_raw h3600
module pthres pmin=1
module variance delta=30
module dejitter delta=100
module linear

Staring xfbdev with:
export TSLIB_CONSOLEDEVICE=none
export TSLIB_FBDEVICE=/dev/fb
export TSLIB_TSDEVICE=/dev/input/ts0
export TSLIB_CONFFILE=/mnt/ts.conf
export TSLIB_PLUGINDIR=/mnt/lib/ts
export TSLIB_CALIBFILE=/usr/local/pointercal

export PATH=/mnt/bin:$PATH
export LD_LIBRARY_PATH=/mnt/lib

./bin/Xfbdev -screen 800x600x16 -mouse tslib,,device=/dev/input/ts0
