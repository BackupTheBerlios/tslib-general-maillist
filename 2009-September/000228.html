> But whenever I use it I need to put some extra pressure to get cursor
> on right location, if I put low pressure on touch screen it read wrong
> co-ordinate.

Sounds like a hardware problem. Resistive touch screen sensors use
contact made between layers to measure a position, it's quite possible
that your sensor is only making partial contact, therefore sending a
value that's in a different place.

I've seen this on a couple of sensor and driver chip combinations,
I've taken the tack that we when we use the resistive sensors we have
to take sloppy positioning and mis-clicks into the user interface
design.

And I'm now of the opinion that if it's not a capacitive touch screen
then we should be designing in some sort of Bluetooth interface we can
drive from an iPhone, because anyone who's used an iPhone or equivalent
device for any amount of time is going to hate the experience of the
resistive sensor.
