On Jan 18, 2010, at 3:52 AM, Andreas Känner wrote:
> 
> I want to use directfb it in conjunction with tslib but it always grabs  
> the standard linux input driver. I think it has to do with this log  
> message: Couldnt load module pthres (see below)

It obeys ts.conf, your ts.conf is most likely requesting the input raw module.  If you want it to use a different mechanism to talk to your ts hardware, you'll have to adjust the ts.conf to use the most appropriate module.

> The files:
> 
> pthres.a
> pthres.la
> pthres.so
> 
> live in a directory named "ts" inside the lib directory. I looks like  
> directfb is not searching for pthres in this directory.
> 
> Is this path adjustable in tslib so that directfb can find it?
> 
> I am glad to hear any suggestions.

All the paths are adjustable via environment variables.  See the README in the source tree for the exact variable names.
