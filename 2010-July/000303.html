Hi all,

I'm using tslib's ts_calibrate and ts_test utilities as part of a test
suite intended for production testing of PCBs.  The test suite interface
is normally via serial console, but the user *can* plug a USB keyboard
and use the LCD as the console.

In the test suite, the ts_calibrate tool is first called, followed by
ts_test.  Now if it's started from the serial console, the tslib suite
inherits that console and ^C works as expected.  However, if the program
is started from ttyX (any of the framebuffer consoles)... ^C does not
work as tslib has grabbed control of /dev/tty.

Now, if the touchscreen is working, there isn't a problem...
ts_calibrate is soon complete, and the latest versions of tslib have a
quit button that you can touch.  However, if the touchscreen is not
functional, the only thing you can do is to do a 'killall ts_calibrate'
or 'killall ts_test' from either SSH or serial console.

Is it possible for ts_calibrate/ts_test to hook, say, the ESC key to
allow the user to be able to terminate the application in cases where
the touchscreen is not operational?
